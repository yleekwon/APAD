INFORMATION

This project had three phases which included:

  1. Developing the basic data model in SQL for storing data and creating python utility functions to manipulate the data
  2. Deploying the application to a Google App Engine's appspot.com site and building a web interface for the application
  3. Developing a functional Android app that can access the website using Android Studio and kotlin

FIRST PHASE
The goal of this phase was to create the framework for an event-planning web application.

POINTS OF INTEREST:
  • Users: These are people who use the application
  • Venues: These are the places where a pickup sport event may be planned
  • Events: These are the events that are currently in play

FUNCTIONS CREATED:
  • Add a user (admin only)
  • Add a new venue (admin only)
  • Start an event (user or admin on behalf of a user)
  • Display timeslot availability
  • Display all venues where a particular timeslot is available
  • List events at a venue given date/time
  • Join an event
  • Remove an event (admin only)

TECHNOLOGIES USED:
  • Jupyter Notebook
  • Sqlite
  • Python

SECOND PHASE
The goal of this phase was to deploy our app onto the web so that it is accessable via a browser. As such, the hosting was done through Google App Engine (GAE).

POINTS OF INTEREST:
  • The site is publically available which was done with GAE and hosted at comehither.appspot.com
  • Google Cloud SQL Link was used for our SQL-based database
  • Wrote python script that handles http requests and provides paths and corresponding classes to act on those paths
  • Create HTML files for each function

TECHNOLOGIES USED:
  • Google App Engine
  • Google Cloud SQL Link
  • Python
  • HTML/CSS/Javascript

THIRD PHASE
The goal of this phase was to create an Android app that allows the user to access the functionality from phases 1 and 2. The app is intended for users and as such doesn't include the admin only functions.

POINTS OF INTEREST:
  • Account Management: Login/Logout
  • App-related Functions: Displays, Join an event, Show events

TECHNOLOGIES USED:
  • Google App Engine
  • Google Cloud SQL Link
  • Python
  • HTML/CSS/Javascript
  • Android Studio
  • Kotlin

HOW TO: WEB INTERFACE

  1. Login Page
    The first page the user arrives to when entering comehither.appspot.com is the login page. Because of the purpose of the design, users are not able to register into the system. Only users that are already inside the database are able to login. The email and EID are both unique and not null and as such, we used them in order to validate the user as they login.
  2. Welcome Page
    Once the user logins, they are entered into a welcome page that informs the user of their admin status. The operations that the user can do are shown in the column on the left.



  3. Form Page
    Every operation starts with a form page that the user can fill out. The necessary parameters are shown in the form and once the submit button is pressed, the user is redirected to either a success page or error page.



  4. Success Page
    If the user has performed the operation succesfully, they are redirected to a success page. The success page states that the action was succesful and a review of the information inputted.



  5. Error Page
    If the user is not an admin and tries to use admin-only operations, they are redirected to the welcome page that includes a warning that they are not admin.



HOW TO: APP INTERFACE

  1. Login Page
    The first fragment the user arrives to is the login page. Similarily to the webpage, there is no register user.



  2. Navigation View
    Once the user logins, they are entered into a navigation fragment that contains a list of the operations the user can do within the app.



  3. Form Fragment
    When the user picks an operation, they are taken to that specific fragment where they are asked to input certain information. Once the submit button is pressed, a textview populates with the information requested. A back button is also included which returns the user to the Navigation fragment.
