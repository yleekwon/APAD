ABOUT

The purpose of this project was to develop a more intuitive UT room booking service that has both a web interface and an Android application. The users are the UT staff and students and would further be seperated into regular users and admin. The admin have special actions that only they can perform.

With this service, a user is able to perform the following actions:

  • Create an event: Users are allowed to create their own events and book venues at a specific time
  • Display availability at a venue: Users are able to see the available times of a specific venue
  • Display available times at all venues: Users are able to see all the available venues at a specific time
  • Display events at a venue: Users are able to see a specific event given a venue and time
An admin can do all of the actions above and more:
  • Add a user: Admins can add users into the database which would allow them to login into this service
  • Add a venue: Admins can create new venues which would allow users to book events in them
  • Remove an user: Admins can remove users from the database which would stop them from using this service
